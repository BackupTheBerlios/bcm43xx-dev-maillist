I fixed the problem by passing -K to dhcpcd, 

 -K, --nolink
   Don't receive link messages for carrier status.  You
   should only have to use this with buggy device drivers or
   running dhcpcd through a network manager.

(I don't think it's an authentication / association problem, but
something else related to this "link message for carrier status",
specifically involving dhcpcd.)
